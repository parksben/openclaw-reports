## 1. Agent Team Orchestration（多智能体团队编排）

**链接：** https://clawhub.ai/arminnaimi/agent-team-orchestration

**功能描述：**
生产级多智能体团队协作框架。定义 Orchestrator（路由调度）、Builder（执行产出）、Reviewer（质量把关）、Ops（定时巡检）四类角色，规范任务状态流转（Inbox → Assigned → In Progress → Review → Done/Failed），内置标准化 Handoff 协议（做了什么、产物在哪、如何验证、遗留问题、下一步行动）。包含 team-setup、task-lifecycle、communication、patterns 四份参考文档，涵盖并行研究、规格→构建→测试、升级处理等经典流程。

**适用场景：**
- 构建多 Agent 协作工作流（如研究 + 编写 + 审核流水线）
- 长期运行的 Agent 团队任务，需要防质量漂移
- 跨 session 多步骤任务状态追踪与交接
- 规范团队内 Agent 角色划分，避免职责重叠导致混乱

## 2. Playwright MCP（Playwright 浏览器自动化）

**链接：** https://clawhub.ai/spiceman161/playwright-mcp

**功能描述：**
基于 Playwright MCP 服务器的浏览器自动化 Skill，支持 Chrome、Firefox、WebKit 三大浏览器。提供 browser_navigate、browser_click、browser_type、browser_evaluate、browser_snapshot 等完整 MCP 工具集，支持无头模式、视口设定、HTTPS 容错、截图/视频录制。可配置 Host 白名单防止误操作，支持沙箱模式与 Service Worker 拦截。

**适用场景：**
- Agent 驱动的 Web 自动化测试（登录、表单填写、数据提取）
- 抓取 JavaScript 渲染页面的动态内容
- 跨浏览器兼容性批量验证
- 构建端到端自动化流水线，替代传统爬虫

## 3. Data Analysis（决策导向数据分析专家）

**链接：** https://clawhub.ai/ivangdavila/data-analysis

**功能描述：**
专为"分析服务决策"设计的数据分析方法论 Skill。核心理念：分析没有决策导向就只是算术。内置完整方法论框架：先锁定决策背景再动手、指标契约先行（实体/粒度/分子分母/时间窗/数据源）、分离提取/转化/解释三层、按问题类型选方法（假设检验、回归、队列分析、分群、异常检测）、所有输出含置信区间+建议行动。内含 metric-contracts、chart-selection、decision-briefs、pitfalls、techniques 五份参考文件。

**适用场景：**
- KPI 指标调试与实验结果解读
- A/B 测试、漏斗分析、用户队列分析
- 高管汇报前的数据自查（避免常见陷阱）
- 数据科学团队的分析规范化与同行评审

## 4. Memory Tiering（三层记忆架构）

**链接：** https://clawhub.ai/sarielwang93/memory-tiering

**功能描述：**
实现动态三层记忆管理：🔥 HOT（当前会话上下文/活跃任务）、🌡️ WARM（用户偏好/系统配置/稳定设置）、❄️ COLD（长期归档/历史决策/里程碑摘要）。提供完整的 Organize-Memory 工作流：摄入 & 审计 → 层级重分配 → 剪枝与摘要 → 校验。可手动触发（"Run memory tiering"/"整理记忆层级"）或 /compact 后自动触发。解决 Agent 上下文压缩后关键信息丢失问题。

**适用场景：**
- 长期运行的 Agent 出现上下文溢出，需要智能分层管理
- 多 session 工作流需要维护热/温/冷三级知识库
- 定期归档历史任务细节，保持 HOT 层精简高效
- 配合 elite-longterm-memory 使用，构建更完整的记忆体系

## 5. Atlassian MCP（Jira & Confluence 集成）

**链接：** https://clawhub.ai/atakanermis/atlassian-mcp

**功能描述：**
通过 MCP 协议将 Jira 和 Confluence 接入 Agent，以 Docker 容器方式运行 mcp-atlassian 服务器，配置 JIRA_URL、JIRA_USERNAME、JIRA_API_TOKEN 即可启动。支持查询 Issue、管理项目、创建任务、交互 Atlassian 全套工具。内置 run_mcp_atlassian.sh 脚本简化部署流程。

**适用场景：**
- 让 Agent 直接读写 Jira Issue，自动更新任务状态
- 结合 coding-agent，实现"分析需求 → 写代码 → 更新 Jira"全链路
- Confluence 文档的自动化写入与查询
- 企业 DevOps 团队将 Agent 嵌入 Jira 工作流
