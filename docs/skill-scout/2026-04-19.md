## 1. GitHub Skill

**作者：** steipete  
**链接：** https://clawhub.ai/steipete/github

### 功能描述

通过 `gh` CLI 与 GitHub 交互的指令集 Skill。无需额外配置，直接调用 `gh` 命令完成 PR 检查、CI 状态查询、Issue 管理等操作。

### 核心能力

- 检查 PR 的 CI 状态：`gh pr checks 55 --repo owner/repo`
- 列出最近 workflow 运行记录
- 查看失败步骤的详细日志
- 使用 `gh api` 进行高级查询，支持 `--json` + `--jq` 过滤输出

### 适用场景

- 日常 PR review 流程中快速查看 CI 结果
- 自动化脚本中需要查询 GitHub 数据
- 不想打开浏览器、直接在终端/Agent 里操作 GitHub

### 安全评级

✅ 无安装步骤，纯指令型 Skill，风险极低

## 2. Weather Skill

**作者：** steipete  
**链接：** https://clawhub.ai/steipete/weather

### 功能描述

无需 API Key，通过 `wttr.in` 和 `Open-Meteo` 两个免费服务获取天气和预报数据。

### 核心能力

- 快速一行命令获取当前天气：`curl -s "wttr.in/Shanghai?format=3"`
- 支持自定义格式（温度、湿度、风速、月相等）
- 支持机场代码查询（如 `wttr.in/PVG`）
- Open-Meteo 作为 fallback，返回 JSON 格式，适合程序化处理

### 适用场景

- Agent 需要感知用户所在地天气时自动调用
- 出行提醒、穿衣建议等场景
- 不想申请 API Key 的轻量天气集成

### 安全评级

✅ 仅发出 HTTP 请求，无凭证，无副作用

## 3. gh-issues — GitHub Issue 自动修复

**作者：** jackhua6  
**链接：** https://clawhub.ai/jackhua6/gh-issues

### 功能描述

强大的 GitHub Issue 自动化处理 Skill。解析 Issue 列表，并行派发子 Agent 自动实现修复、提交代码、开 PR，支持 Fork 模式和 PR Review 自动响应。

### 核心能力

- 自动获取 Issue 列表，支持按 label/milestone/assignee 过滤
- 并行派发最多 8 个子 Agent 同时修复不同 Issue
- 自动创建分支、提交代码、开 PR
- `--watch` 模式持续轮询新 Issue
- `--cron` 模式支持定时任务，每次处理一个 Issue，防止重复
- Phase 6 自动处理 PR Review 评论，派发子 Agent 修复并回复

### 适用场景

- 开源项目维护者批量处理 bug 类 Issue
- 团队内部自动化修复流水线
- 配合 OpenClaw cron 实现每日自动修复

### 安全评级

⚠️ 会向 GitHub 推送代码并开 PR，使用前确认 GH_TOKEN 权限范围

## 4. Health Tracker Skill

**作者：** stellarhold170nt  
**链接：** https://clawhub.ai/stellarhold170nt/healthcheck

### 功能描述

轻量级健康数据追踪 Skill，用 JSON 文件记录饮水量和睡眠时间，纯 Node.js 内置模块实现，无任何外部依赖。

### 核心能力

- 记录饮水：说"喝了 2 杯水"自动写入
- 记录睡眠/起床时间，自动计算睡眠时长
- 查看今日统计（饮水杯数、睡眠记录数）
- 支持修改和删除最近一条记录

### 适用场景

- 个人健康习惯追踪
- 配合 Agent 的日常 check-in 流程
- 不想用第三方 App、数据完全本地存储

### 安全评级

✅ 纯本地文件操作，无网络请求，无隐私风险

## 5. Skill Creator

**作者：** chindden  
**链接：** https://clawhub.ai/chindden/skill-creator

### 功能描述

专门用于创建和优化 AgentSkill 的元 Skill。提供完整的 Skill 设计方法论，包括结构规范、渐进式披露设计原则、脚本/引用/资源文件的组织方式。

### 核心能力

- 6 步 Skill 创作流程：理解 → 规划 → 初始化 → 编写 → 打包 → 迭代
- 渐进式披露设计：metadata（~100词）→ SKILL.md → 引用文件，按需加载
- 提供 `init_skill.py` 和 `package_skill.py` 脚本自动化初始化和打包
- 详细的 SKILL.md 写作规范（frontmatter、body、触发描述）
- 多种组织模式：按域分文件、按框架分文件、条件加载

### 适用场景

- 想为自己的工作流创建专属 Skill
- 审查和优化现有 Skill 的结构
- 学习 AgentSkill 最佳实践

### 安全评级

✅ 纯指令型 Skill，无外部调用
