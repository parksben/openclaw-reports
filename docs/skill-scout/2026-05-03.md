## 1. Slack（Slack 消息管理集成）

**链接：** https://clawhub.ai/steipete/slack

**功能描述：**
@steipete 出品的 Slack 集成 Skill，通过 Clawdbot 的 Bot Token 与 Slack 深度整合。支持发送/编辑/删除消息、Emoji 反应（react/list）、消息置顶/取消置顶、读取频道历史消息、查询成员信息、列出自定义 Emoji。操作通过结构化 JSON 参数驱动，支持 `channel:` 和 `user:` 两种消息投递目标。

**适用场景：**
- Agent 自动向 Slack 频道推送任务完成通知或状态报告
- 用 ✅ Emoji 反应标记已处理消息，代替人工确认
- 置顶重要决策或周报到频道，方便团队快速查阅
- 与 Jira/ClickUp 等工具集成，实现"更新任务 → 通知 Slack"全链路自动化

## 2. n8n Workflow Automation（健壮 n8n 工作流设计）

**链接：** https://clawhub.ai/kowl64/n8n-workflow-automation

**功能描述：**
专为设计和输出生产级 n8n 工作流 JSON 的 Skill。核心特点：内置幂等性设计（防止重复执行时重复写入）、完整错误处理与指数退避重试、每次运行自动生成 run_id 并记录状态行、失败条目写入人工审核队列（HITL Review Queue）、支持生成配套 runbook.md 运维文档。支持 Cron、Webhook、手动三种触发模式。

**适用场景：**
- 构建每日/每周定时汇总报告并发送邮件或上传 Drive
- Webhook 接收外部事件后路由处理，失败自动写入审核队列
- 给现有 n8n 流程补充重试机制和人工审核节点
- 需要审计日志的合规性工作流（如财务对账、权限变更记录）

## 3. ClickUp MCP（官方 ClickUp MCP 全功能集成）

**链接：** https://clawhub.ai/pvoo/clickup-mcp

**功能描述：**
通过 ClickUp 官方 MCP 服务器接入 ClickUp 工作区，提供 32 个工具覆盖完整 ClickUp 功能：全局搜索（跨任务/文档/仪表板/聊天/文件）、任务创建/更新/获取、附件上传、标签管理、评论（支持 @提及）、时间追踪（开始/停止计时器）、Chat 消息、文档管理等。支持 OAuth 直连（Claude Code/Cursor/Windsurf）或 mcporter 中间层方式，Token 有效期约 10 年。

**适用场景：**
- 让 Agent 自动读写 ClickUp 任务，省去手动更新状态
- 结合 coding-agent：代码合并后自动将对应 ClickUp 任务更新为 Done
- 跨 Space/List/Folder 的全局搜索，快速定位历史任务和文档
- 自动追踪工时：Agent 开始执行任务时自动启动计时器

## 4. WordPress MCP（AI 全面管理 WordPress 站点）

**链接：** https://clawhub.ai/jordymeow/wordpress-mcp

**功能描述：**
通过 AI Engine WordPress 插件暴露的 MCP Server，让 Agent 以 JSON-RPC 2.0 方式管理 WordPress 站点。默认开放 WordPress 核心功能（文章、页面、评论、用户、媒体、分类、设置），可选开启插件管理、主题管理、数据库 SQL 查询、WooCommerce（商品/订单/客户）、SEO Engine（SEO 扫描/分析）、Social Engine（社交媒体排期）等模块。开箱即用，免费插件。

**适用场景：**
- Agent 自动发布/更新博客文章，批量修改 SEO 元信息
- WooCommerce 店铺运营：自动同步商品信息、处理订单查询
- 定期执行 SEO 健康扫描，找出需要优化的页面并生成报告
- 与社交媒体 Skill 联动：文章发布后自动排期社交媒体推广

## 5. Vibe Coding（AI 辅助编程方法论指南）

**链接：** https://clawhub.ai/ivangdavila/vibe-coding

**功能描述：**
由 Andrej Karpathy（2025年2月）提出的 Vibe Coding 方法论完整指南。@ivangdavila 整理了核心规则、提示词技巧（prompting.md）、研究-计划-实现工作流（workflow.md）、Rules 文件模板（.cursorrules/CLAUDE.md/.windsurfrules）、常见陷阱与修复方案（pitfalls.md）、工具选型指南（tools.md）。核心理念：用自然语言描述目标让 AI 生成代码，在验收结果而非逐行阅读代码。

**适用场景：**
- 非专业开发者快速构建原型，跳过繁琐语法学习
- 专业开发者提升 coding-agent 使用效率，避免常见 Vibe Coding 陷阱
- 为 Cursor/Claude Code/Windsurf 编写高质量 Rules 文件
- 团队统一 AI 辅助开发规范，区分"可以 vibe"和"必须审查"的代码边界
